# 操作手冊(給非技術人員)

> 這份手冊寫給**不懂程式的操作者**:每天要按哪些按鈕、看哪些數字、遇到紅字怎麼辦。
> 全程只需要瀏覽器,不需要碰任何程式碼。
> (技術設定與上線準備是工程師的事,見《上線作戰手冊》README.md)

---

## 這套系統是什麼(30 秒版)

它幫公司**開發美國買家**:你把潛在客戶名單餵進去,AI 會上網研究每一家公司、
打分數排優先序、寫好三封英文開發信(第一封打招呼、四天後補一封、六天後收尾),
**你看過按一次「核准」**,系統就照節奏自動寄出;對方一回信,系統自動停止後續、
提醒你接手談生意。

你的角色只有兩個:**品質把關**(信寄出前看一眼)和**談生意**(對方回信後接手)。

---

## 一、開機與關機

**開機**(電腦重開機後做一次):

1. 打開「終端機」App(Terminal)
2. 貼上這兩行,按 Enter:
   ```
   cd ~/ankomn/buyer-intelligence && source .venv/bin/activate
   buyer-intel serve
   ```
3. 看到一個框框寫著 `http://localhost:8000` 就成功了
4. 打開瀏覽器,網址列輸入 **`localhost:8000`**

**這個終端機視窗不要關**——寄信的排程器住在裡面。視窗開著,信才會照排程寄;
關掉或闔上筆電,寄送就暫停(不會弄丟,重開機後接著寄)。

**關機**:回到終端機視窗按 `Ctrl + C` 即可。

## 二、七個頁面,各是幹嘛的

| 頁面 | 一句話 | 你多常去 |
|---|---|---|
| **儀表板** | 全局概況:漏斗、逾期警示、快速入口 | 每天第一眼 |
| **名單** | 所有潛在買家:分數、階段、聯絡人 | 查某一家時 |
| **覆核佇列** | ⭐ 等你核准的信(三封一組) | 每天,核心工作 |
| **寄送佇列** | ⭐ 寄信中控台:排了什麼、寄了沒、對方回了沒 | 每天掃一眼 |
| **匯入名單** | 上傳新名單(CSV 或地圖掃描) | 每週一次 |
| **Pipeline** | 讓 AI 研究新名單、寫信(按一顆按鈕) | 每週,匯入後 |
| **名片掃描** | 展會用:手機拍名片直接建檔 | 只有展期 |

## 三、每天的例行工作(10 分鐘)

### ① 開儀表板看兩個數字
- **「⚠️ 逾期未跟進」有東西?** → 點進去,該回信回信、該追樣品追樣品
- **「✉️ 覆核佇列(N)」有數字?** → 進行第 ② 步

### ② 覆核信件(最重要的工作)

進「覆核佇列」,每一家會顯示**三封信**(第一封 + 兩封自動跟進)。你要看的:

1. **「為什麼找上這家」那句是不是真的**——AI 只准用查到的事實,但你是最後防線。
   看到「貴店在賣 Fellow Atmos」這種話,想一下合不合理
2. **收件人對不對**——點旁邊的「🔍 LinkedIn 在職查核」花 30 秒確認這個人還在職
3. 想改就直接在框框裡改字(三封都可以改)

然後二選一:
- **✅ 核准整串** → 三封信排入寄送佇列,之後全自動,你不用再管這家
- **✘ 退回** → 這組信作廢(之後重跑 Pipeline 會產新的)

> 核准後反悔?到「寄送佇列」找到那幾列,按「取消」即可(只要還沒寄出)。

### ③ 掃一眼寄送佇列
- 「今日寄出 X/Y」正常累積 → 沒事
- 有 **✘ 失敗** 的列 → 按「重排」讓它再試;一直失敗就截圖找工程師
- 看到 **📩 對方已回信** → 進行第四章

**這頁會自己更新,不用重新整理。**

---

## 四、買家回信了!(最開心的時刻)

系統偵測到回信會自動做三件事:停掉後續跟進信(不會騷擾對方)、把該公司標成
「已跟進」、在寄送佇列日誌顯示 📩。**接下來換你上場:**

1. **去 Gmail 讀回信,親自回覆**——從這裡開始是真對話,系統不代筆
2. 談出進展後,到「名單」找到那家公司,點進詳情頁,**按對應的階段按鈕**:

| 情況 | 按這顆 | 之後 |
|---|---|---|
| 敲定會議/通話 | 📅 約到會議 | 7 天後提醒你跟進 |
| 樣品寄出去了 | 📦 樣品寄出 | 7 天後提醒追試用回饋 |
| 報價單發了 | 💰 報價中 | 5 天後提醒追決策 |
| **收到訂單!** | 🎉 收到 PO | 進漏斗終點 🎊 |
| 對方明確沒興趣 | 🗑 歸檔 | 結案 |

每次按按鈕,儀表板的漏斗就會前進一格——老闆看漏斗就知道戰況,你不用寫週報。

> **對方回信說 UNSUBSCRIBE / 不要再寄?** 什麼都不用做,系統自動把他加入
> 退訂名單、永不再寄。你也可以在寄送佇列頁底部手動加。

## 五、每週的例行工作(30 分鐘)

### ① 補充新名單(匯入頁)
- **有 CSV 檔**(例如 Apollo 匯出):上傳 → 選來源標籤 → 看預覽勾選 → 確認匯入
- **想掃某個城市的店家**:在「Google 地圖掃描」輸入例如
  `specialty coffee roaster in Portland, OR` → 掃描並入庫

### ② 讓 AI 開工(Pipeline 頁)
- 「筆數上限」填 **5–10**(一次別貪多,每筆要跑 2–4 分鐘)
- 按「▶ 開始執行」→ 可以關頁面去做別的事
- 跑完後「覆核佇列」就會有新信等你(回到每天的第 ② 步)

## 六、看到這些畫面,是什麼意思

| 你看到 | 意思 | 要做什麼 |
|---|---|---|
| 佇列狀態「⏳ 排程中」 | 信寫好了,等排定時間到 | 不用做,想反悔按「取消」 |
| 佇列狀態「✅ 已寄出」 | 真的寄出去了 | 不用做 |
| 「🛑 今日已達寄出上限」 | 每天限量是**故意的**(保護寄件信箱不被 Gmail 當垃圾信) | 不用做,明天自動繼續 |
| 「⏲ 節奏護欄」 | 兩封信之間故意隔一小時(像真人,不像機器) | 不用做 |
| 「⏸ 沒有到期信件」 | 排程器正常巡邏中,現在沒事做 | 不用做 |
| 紅字「排程器已暫停」 | 有人按了暫停 | 確認沒事後按「▶ 恢復自動寄送」 |
| 紅字「sender.address 未填」 | 公司地址沒設定(法規必填) | 找工程師 |
| 一列「✘ 失敗」 | 那封信沒寄成功 | 按「重排」再試;連續失敗找工程師 |

## 七、三條鐵則(違反會出事)

1. **沒看過的信不要核准**——你是 AI 幻覺的最後防線。寧可退回,不要放行有疑慮的信
2. **不要繞過系統手動狂寄**——每天限量、間隔一小時都是防 Gmail 封鎖的保護,
   嫌慢是正常的,cold email 就是慢工
3. **退訂名單永遠不要刪**——對方說過不要寄,再寄就是違法(美國 CAN-SPAM 罰款
   每封最高五萬美元)

## 八、緊急情況

| 情況 | 動作 |
|---|---|
| **「先全部停下來!」** | 寄送佇列頁 → 按「⏸ 暫停自動寄送」(已排的信保留,恢復後繼續) |
| 寄錯人了 | 已寄出的收不回(跟平常寄 email 一樣);該 lead 詳情頁按 🗑 歸檔,剩餘未寄的會自動取消 |
| 頁面打不開 | 看終端機視窗還在不在;不在就照第一章重新開機 |
| 其他任何異常 | 截圖「寄送佇列」頁的日誌區,傳給工程師 |
